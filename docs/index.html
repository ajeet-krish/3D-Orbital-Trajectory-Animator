Two-Body Problem & Keplerian Orbits

Physics of Orbital Motion

A satellite stays in orbit because of a balance between its forward motion and Earth's gravity pulling it inward. This is the two-body problem: two point masses interacting through gravity, with no other forces at play. Newton's law of universal gravitation describes the force:

$$ \mathbf{F} = -\frac{GMm}{r^2} \hat{\mathbf{r}} = m\mathbf{a} $$
Newton's law of gravitation, the central force that governs orbital motion

From this inverse-square law, you can show that the satellite traces a conic section: an ellipse for orbits that stay bound to Earth, and a parabola or hyperbola for trajectories that escape. This is Kepler's First Law.

Kepler's equation connects time to position along the orbit. It relates the mean anomaly $M$ (which increases steadily with time) to the eccentric anomaly $E$, which then gives the true anomaly $\nu$ (the actual angular position):

$$ M = E - e \sin E \qquad \tan\frac{\nu}{2} = \sqrt{\frac{1+e}{1-e}} \tan\frac{E}{2} $$
Kepler's equation (left) and how eccentric anomaly relates to true anomaly (right)

This project uses PyAstronomy's KeplerEllipse to solve Kepler's equation numerically, then animates the resulting trajectory with Matplotlib.

Classical Orbital Elements

Every orbit can be described by six numbers, known as the classical orbital elements. Think of them as the address of a satellite in space:

Element Symbol Description
Semi-major axis $a$ Size of the orbit (km or AU)
Eccentricity $e$ Shape: $0$ = circular, $0 < e < 1$ = elliptical
Inclination $i$ Tilt relative to reference plane (degrees)
RAAN $\Omega$ Longitude of the ascending node (degrees)
Arg. of periapsis $\omega$ Orientation of the ellipse in its orbital plane (degrees)
True anomaly $\nu$ Current position of the satellite along the orbit (degrees)

Vis-Viva Equation

The Vis-Viva equation tells you a satellite's speed at any point in its orbit. It only needs the gravitational parameter $\mu = GM$ and your current distance $r$ from Earth:

$$ v^2 = \mu \left( \frac{2}{r} - \frac{1}{a} \right) $$
Vis-Viva equation: conservation of energy applied to orbital mechanics

At periapsis (closest approach), $r$ is smallest and $v$ is highest. At apoapsis (farthest point), the reverse happens. This is Kepler's Second Law: a satellite sweeps equal areas in equal times, so it moves faster when closer and slower when farther.

Core Functions

compute_orbit(): generate 3D orbit positions from Keplerian elements 
def compute_orbit(
    a=1.0, per=1.0, e=0.0, Omega=0.0, i=0.0, w=0.0, n_orbits=2, n_points=300
):
    ke = pyasl.KeplerEllipse(a=a, per=per, e=e, Omega=Omega, i=i, w=w)
    t = np.linspace(0, n_orbits * per, n_points)
    pos = ke.xyzPos(t)
    return pos, t, ke
PyAstronomy's KeplerEllipse solves Kepler's equation internally; xyzPos returns N×3 position array in Earth-radii units.
draw_earth(): wireframe Earth sphere at origin 
def draw_earth(ax, radius=1.0, color="#ff79c6", linewidth=0.6, alpha=0.6):
    n_theta = 20; n_phi = 20
    theta = np.linspace(0, 2 * np.pi, n_theta)
    phi = np.linspace(-np.pi / 2, np.pi / 2, n_phi)

    for i in range(0, n_phi, 3):
        x = radius * np.cos(phi[i]) * np.cos(theta)
        y = radius * np.cos(phi[i]) * np.sin(theta)
        z = radius * np.sin(phi[i]) * np.ones_like(theta)
        ax.plot(x, y, z, color=color, linewidth=linewidth, alpha=alpha)

    for i in range(0, n_theta, 3):
        x = radius * np.cos(phi) * np.cos(theta[i])
        y = radius * np.cos(phi) * np.sin(theta[i])
        z = radius * np.sin(phi)
        ax.plot(x, y, z, color=color, linewidth=linewidth, alpha=alpha)
Subsampled latitude/longitude lines every 3rd grid line for visual clarity.
setup_3d_axes(): dark-themed 3D axis configuration 
def setup_3d_axes(ax, pos):
    limit = pos[:, :3].max() * 1.4
    ax.set_xlim([-limit, limit])
    ax.set_ylim([-limit, limit])
    ax.set_zlim([-limit, limit])
    ax.set_box_aspect([1, 1, 1])
    ax.set_xlabel("X (Earth Radii)")
    ax.set_ylabel("Y (Earth Radii)")
    ax.set_zlabel("Z (Earth Radii)")
    ax.xaxis.pane.fill = False
    ax.yaxis.pane.fill = False
    ax.zaxis.pane.fill = False
    ax.xaxis.pane.set_edgecolor("#3b4261")
    ax.yaxis.pane.set_edgecolor("#3b4261")
    ax.zaxis.pane.set_edgecolor("#3b4261")
    ax.tick_params(colors="#565f89")
Consistent dark-theme styling for all 3D plots. Limits auto-scaled with 40% padding.

Low Earth Orbit (LEO): Single Orbit Demonstration

A Low Earth Orbit is roughly circular with low eccentricity ($e \approx 0.01$ to $0.05$) and altitudes between 200 and 2000 km. The ISS lives here. This demo uses dimensionless units where $a = 1.0$ equals roughly one Earth radius (~6371 km).

What the Annotations Mean

  • Inclination $i = 30^\circ$: the orbit is tilted compared to the equator
  • RAAN $\Omega = 0^\circ$: the ascending node lines up with the reference direction
  • Argument of periapsis $\omega = 0^\circ$: periapsis sits on the line of nodes
  • True anomaly $\nu$: where the satellite is along its path at any moment (animated)

The wireframe grid shows Earth with latitude and longitude lines. The pink arc and dotted line mark the inclination angle and the line of nodes.

Animated LEO with Velocity Vector

Arrow3D: custom 3D arrow for velocity vector display 
class Arrow3D(FancyArrowPatch):
    def __init__(self, xs, ys, zs, *args, **kwargs):
        super().__init__((0, 0), (0, 0), *args, **kwargs)
        self._verts3d = xs, ys, zs

    def do_3d_projection(self):
        xs3d, ys3d, zs3d = self._verts3d
        xs, ys, zs = proj3d.proj_transform(xs3d, ys3d, zs3d, self.axes.M)
        self.set_positions((xs[0], ys[0]), (xs[1], ys[1]))
        return min(zs)

animate_leo_with_velocity(): animation loop with trail + velocity 
def animate_leo_with_velocity():
    pos, t, ke = compute_orbit(
        a=1.0, per=1.0, e=0.05, i=30.0, Omega=0.0, w=0.0,
        n_orbits=2, n_points=400
    )
    fig = plt.figure(figsize=(10, 8))
    ax = fig.add_subplot(111, projection="3d")
    setup_3d_axes(ax, pos)
    draw_earth(ax, radius=1.0, color="#ff79c6", alpha=0.5)
    ax.plot(pos[:, 0], pos[:, 1], pos[:, 2],
            color="#3b4261", linewidth=0.8, alpha=0.4, label="Full orbit")

    trail_len = 50
    dt = t[1] - t[0]

    def update(frame):
        sat.set_data([pos[frame, 0]], [pos[frame, 1]])
        sat.set_3d_properties([pos[frame, 2]])
        start = max(0, frame - trail_len)
        trail.set_data(pos[start:frame+1, 0], pos[start:frame+1, 1])
        trail.set_3d_properties(pos[start:frame+1, 2])
        if frame > 1:
            v = (pos[frame] - pos[frame-1]) / dt
            v_norm = v / (np.linalg.norm(v) + 1e-12)
            tip = pos[frame] + v_norm * 0.3
            vel_arrow.set_positions(pos[frame], tip)
            vel_arrow.set_visible(True)
        return sat, trail, vel_arrow

    anim = animation.FuncAnimation(fig, update, frames=len(pos),
                                   interval=30, repeat=True)
    return anim
The velocity vector is computed via finite difference; the trail uses a fixed-length sliding window of recent positions.

Hohmann Transfer: Moving Between Orbits

The Hohmann transfer is the most fuel-efficient way to move a spacecraft between two circular orbits in the same plane. You fire the engine twice: once to stretch the orbit into an ellipse, and again to circularize at the target altitude.

In this demo, a satellite starts in a low orbit ($r \approx 1.1$ Earth radii), fires its engine at perigee to enter an elliptical transfer orbit whose apogee reaches the target radius ($r \approx 2.5$). At apogee, a second burn circularizes the orbit. The animation labels each phase.

Two-burn Hohmann transfer: burn 1 at perigee (right side) injects onto the transfer ellipse; burn 2 at apogee (left side) circularizes. Each burn labeled as the spacecraft passes through.
Hohmann transfer construction 
r1, r2 = 1.1, 2.5
a_t = (r1 + r2) / 2          # transfer semi-major axis
e_t = (r2 - r1) / (r2 + r1)  # transfer eccentricity

pos_a, _, _ = compute_orbit(a=r1, e=0, i=30, n_orbits=1, n_points=120)
pos_t, _, _ = compute_orbit(a=a_t, e=e_t, i=30, w=0, n_orbits=1, n_points=200)
pos_b, _, _ = compute_orbit(a=r2, e=0, i=30, n_orbits=1, n_points=120)

# Concatenated flight path: half of A → full transfer → half of B
path = np.vstack([pos_a[:60], pos_t, pos_b[:60]])
The transfer orbit shares a perigee with the lower orbit and an apogee with the higher orbit. Concatenating the three segments creates a seamless flight path.

Molniya Orbit: High-Latitude Communications

The Molniya (named after the Russian word for “lightning”) is a highly elliptical orbit designed to serve places near the poles:

  • High eccentricity $e \approx 0.74$: a big difference between its farthest and closest points to Earth
  • Critical inclination $i = 63.4^\circ$ (or $116.6^\circ$ for retrograde orbits)
  • Argument of perigee $\omega = 270^\circ$: puts the apogee at the northernmost point

Why $63.4^\circ$?

At this specific critical inclination, something useful happens: the orbit stops twisting. Normally, Earth's bulge (the $J_2$ effect) causes an orbit's high point to drift over time. But at $63.4^\circ$, that drift drops to zero. The apogee stays locked over the northern hemisphere. For a communications satellite serving Russia, Canada, or Scandinavia, this means several hours of continuous coverage per orbit at high elevation angles, something GEO satellites can't do since they're stuck over the equator.

Why this matters: Without the critical inclination, the apogee would slowly drift away from the target region. At $63.4^\circ$, this drift cancels out. It is a passive stabilization mechanism.

J2 Precession: What Happens at Other Inclinations

Earth is not a perfect sphere. Its equatorial bulge (called the $J_2$ perturbation) exerts a torque on any inclined orbit. This causes the orbital plane to slowly rotate around Earth's axis (RAAN precession) and the ellipse's orientation to swing within its plane (apsidal precession).

The animation below shows both effects side by side. On the left, RAAN precession rotates the whole orbit around the polar axis. On the right, apsidal precession swings the perigee around the orbit. At $i = 63.4^\circ$, the apsidal drift rate hits zero: the perigee stops moving. That is why Molniya uses this critical inclination.

Left: RAAN precession: orbital plane rotates around Earth's axis. Right: apsidal precession: perigee drifts within the orbital plane (shown for $i=30^\circ$, where drift is nonzero).

Apogee Dwell

Kepler's Second Law says a satellite sweeps equal areas in equal times. Near apogee (the farthest point), the satellite is at its slowest. A Molniya satellite can spend 8 to 11 hours per 12-hour orbit over the northern hemisphere, then whip quickly through perigee over the southern hemisphere. This long dwell time means a constellation of just three Molniya satellites can provide continuous 24/7 coverage of high-latitude regions.

Molniya orbit: note the slow motion near apogee (top) and fast sweep through perigee (bottom). The orange callout box highlights the dwell characteristic.

05

Orbit Regime Comparison

Different missions need different kinds of orbits. The parameters ($a$, $e$, $i$) are chosen to balance coverage area, revisit time, launch cost, and communication delay. Here are four standard orbit types, each designed for a specific job.

LEO
Low Earth Orbit
$a$ 1.1
$e$ 0.02
$i$ 30°
Used for Earth observation and the ISS. Fast revisit times, high resolution, low latency. The most common type of orbit.
GEO
Geostationary
$a$ 1.0
$e$ 0.00
$i$ 0°
Used for communications and weather satellites. Stays fixed over one spot on the equator, great for continuous regional coverage.
Polar
Polar Orbit
$a$ 1.1
$e$ 0.01
$i$ 90°
Used for reconnaissance, mapping, and weather. The orbital plane stays fixed while Earth rotates underneath, eventually covering every point on the globe.
Molniya
Molniya Orbit
$a$ 1.0
$e$ 0.74
$i$ 63.4°
Used for high-latitude communications (Russia, Canada). Highly elliptical, so the satellite hovers for hours over northern regions near apogee.

Here is the key idea: inclination determines which latitudes a satellite can see. Equatorial orbits ($i=0°$) stay over the equator. Polar orbits ($i=90°$) eventually pass over every point on Earth. Highly inclined elliptical orbits ($i=63.4°$) can spend most of their time over high latitudes. Eccentricity controls where the satellite lingers. In a highly elliptical orbit, the satellite moves fast at perigee and slows down at apogee, creating a long dwell time over the apogee region.

Ground Track Projection

A ground track maps the satellite's position onto Earth's surface over time. The 3D orbit positions are projected onto a 2D map, accounting for Earth's rotation beneath the orbital plane. This is how mission planners determine coverage areas and revisit times.

The animation below shows ground tracks for LEO, GEO, and Molniya side by side with their 3D views. Watch how the GEO ground track stays fixed at one point on the equator, the LEO track traces a sinusoidal wave that drifts westward, and the Molniya track loops at high northern latitudes.

Left: 2D ground tracks (lat vs lon) with Earth rotation. Right: corresponding 3D orbits. GEO stays fixed; LEO makes wavy passes; Molniya loops at high latitudes.

Parameter Sweep: Eccentricity & Inclination

This animation changes two orbital parameters over time so you can see how the orbit shape and orientation evolve:

  • Eccentricity $e$: goes from $0.02$ to $0.80$, stretching a near-circle into a highly elongated ellipse ($a = 1.0$, $i = 30^\circ$)
  • Inclination $i$: goes from $0^\circ$ to $90^\circ$, tilting the orbital plane from flat to polar ($a = 1.0$, $e = 0.80$)

The current parameter value appears in the top-left corner as the animation runs. Each orbit has a fading trail and a soft glow behind it.

Phase 1: $e$ sweeps from circular to highly elliptical. Phase 2: $i$ sweeps from equatorial to polar. Parameter value shown in pink at top-left.
Parameter sweep construction 
n_frames = 120
n_pts = 200

# Phase 1: eccentricity sweep 0.02 → 0.80
ecc_values = np.linspace(0.02, 0.80, 60)
# Phase 2: inclination sweep 0° → 90°
incl_values = np.linspace(0, 90, 60)

orbits = []
for e in ecc_values:
    pos, _, _ = compute_orbit(a=1.0, e=e, i=30.0, Omega=0, w=0,
                              n_orbits=1, n_points=n_pts)
    orbits.append(pos)
for i_val in incl_values:
    pos, _, _ = compute_orbit(a=1.0, e=0.80, i=i_val, Omega=0, w=0,
                              n_orbits=1, n_points=n_pts)
    orbits.append(pos)

# Neon glow effect: wide dim line behind the crisp orbit
(glow,) = ax.plot([], [], [], color="#8be9fd", linewidth=5, alpha=0.15)
(orbit_line,) = ax.plot([], [], [], color="#8be9fd", linewidth=2, alpha=0.9)

def update(frame):
    pos = orbits[frame]
    glow.set_data(pos[:, 0], pos[:, 1])
    glow.set_3d_properties(pos[:, 2])
    orbit_line.set_data(pos[:, 0], pos[:, 1])
    orbit_line.set_3d_properties(pos[:, 2])
    # Update parameter label
    if frame < 60:
        param_label.set_text(f"e = {ecc_values[frame]:.2f}")
    else:
        param_label.set_text(f"i = {incl_values[frame-60]:.0f}°")
    return glow, orbit_line, trail, sat, param_label
Two-phase sweep: 60 frames for $e$, 60 frames for $i$. The neon glow layer adds visual depth.

Summary & Takeaways

The full Python source code for everything on this page is available on GitHub. Core orbital mechanics functions live in the orbital package (src/orbital/). Animation generators in scripts/ import from the package and export GIFs.

What the Package Does

  • Solves the two-body problem using PyAstronomy.KeplerEllipse
  • Renders 3D orbits with labeled Keplerian elements
  • Animates satellite motion with fading trails and velocity vectors
  • Demonstrates Hohmann transfers with labeled burn events
  • Shows J2 precession: RAAN drift and apsidal precession side by side
  • Projects ground tracks onto a 2D latitude/longitude map
  • Compares four orbit regimes: LEO, GEO, Polar, and Molniya
  • Sweeps orbital parameters to show how $e$ and $i$ change the orbit
